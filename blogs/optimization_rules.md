# Kernel 优化规则

## 流程规则
1. **每次只进行一项改动** — 不要同时改多个东西，方便定位性能变化原因
2. **记录每次改动** — 改动内容和性能变化写入 `blogs/perf_log.md`
3. **绘制性能变化图表** — 用图表追踪 kernel 性能随优化轮次的变化
4. **只改 `kernels/` 下的文件** — 除非评测代码本身有 bug
5. **每次优化控制在 10 分钟内** (除编译时间，编译最长 20min)
6. **每次完成优化后 print 性能结果**
7. **必须保证正确性** — 每次改动后先跑正确性测试，通过后再跑性能测试
8. **有效优化需要 commit** — 每次有效的 round 都需要保存并 commit 代码
9. **必要时进行联网搜索优化思路** — 连续 3 轮无法取得有效进展后，需要联网搜索新的优化思路

## 每次 Claude 启动时
1. 读取本文件 (`blogs/optimization_rules.md`)
2. 读取性能记录 (`blogs/perf_log.md`)
3. 基于记录继续下一轮优化

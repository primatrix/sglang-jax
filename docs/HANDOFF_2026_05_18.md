## 重要陷阱(必读,避免重蹈覆辙)

### 1. 不能简单去 draft_extend_for_prefill 的 real_bs slice
**Phase 3.6 教训**(代码已有注释 + Phase 4.2 已正确修复):

模型 EXTEND 返回 logits shape = padded_bs(Phase 2 padded 了 logits_indices),不是 real_bs。如果 capture 端不切到 real_bs,scheduler concat 两个 prefill 的 spec_info 时会算成 `1 + 8 = 9`(本应 `1 + 7 = 8`),然后 draft model rotary_emb 报 ShardingTypeError。

**正确模式**(Phase 4.2 已应用):capture/gather 输入保持 padded(命中 precompile bucket)→ 输出 `jax.lax.dynamic_slice_in_dim` 切到 real_bs(scheduler concat 正确)。同模式 Phase 4.3 应用到 draft_extend_for_decode select_index。

### 2. JAX 0.7 sharded array __getitem__ 切片
原代码 `arr[:real_bs, :]` 在 JAX 0.7 sharded array 上报 `ShardingTypeError`,必须用 `jax.lax.dynamic_slice_in_dim(arr, 0, real_bs, axis=0)`。Phase 3.1 commit 已 fix。

### 3. spec_info 字段必须显式 NamedSharding
np.pad 后的 numpy 数组上 device 时默认 sharding 是 `Explicit('data','tensor')`,但 precompile dummy 用 `jnp.ones` 给的是 `{}`(空)。两者不同 cache key。Phase 4.1 修:padding_for_decode 末尾 + precompile dummy 都显式 `device_put(arr, NamedSharding(mesh, P()))`。

### 4. 模型实际 forward shape ≠ 你以为的形状
诊断时必须 print 实际 `forward_batch.input_ids.shape / seq_lens.shape / spec_info.*.shape`,不要靠脑补。Phase 3.6 我以为 model 返回 real_bs,实际返回 padded_bs。

### 5. closest-key 报告 ≠ 实际 miss
JAX `TRACING CACHE MISS` 日志里 "Several previous keys are closest" 是用来辅助调试的,**closest key 可能和 actual matching key 不是同一个**。验证「真 miss vs 报告噪音」需要 register_variant_if_new 二次确认。

### 6. pod 上的 sgl_jax 是 editable install
`pip show sglang-jax` 显示 `Editable project location: /tmp/sglang-jax-baseline/python`。所以 `kubectl cp` 单个文件就能改代码,无需 reinstall。

## 设计决策(已锁定,不可改)

1. **custom_mask 单桶 padding**:不按桶切,固定 = `max_bs × draft_tokens × (max_token + draft_tokens)`。详 ModelRunner._compute_spec_custom_mask_bucket。
2. **RUNTIME 独立遍历**:精度优先用「跑一次完整 spec 链路 + 内层 seq_len 遍历」,**不**手写每个 jit 的 dummy 调用(Phase 3.3 选了这条路径)。
3. **无 reserved padding page**:padding 行 page_indices 复用 batch 内首个真实 page(safe gather)。
4. **L5 StubModelRunner 暂未敲定**:目前直接用真模型(Qwen3-32B)测,L4 stub 测试 backlog。
5. **本期只验 greedy**:非 greedy 采样(top-p/top-k)precompile 推迟 Phase 5.1。
6. **Precompile 启动耗时本期不优化**:5~10 min 可接受。
7. **Spec EXTEND bs 对齐**:方案 B(用户传几个桶 trace 几档 + runtime pad_to_bucket ceil)。已实施。
